📋 Overview

BugWizard is a sophisticated AI-powered web application designed to revolutionize bug reporting and retest workflows in Azure DevOps (ADO) and Atlassian JIRA. Operating in two powerful modes—Developer's Nightmare Mode (Bug Creator) and Developer's Chill Mode (Retester)—it automates the entire process from parsing markdown-formatted bug reports to creating work items and submitting retest comments via Azure DevOps REST API or JIRA REST API.

🚀 Getting Started

System Requirements

  • Modern web browser (Chrome, Edge, Firefox, or Safari recommended)
  • Internet connection for Azure DevOps or JIRA API access
  • Azure DevOps Personal Access Token (PAT) OR JIRA API Token with appropriate permissions

Accessing BugWizard

  1. Open the index.html file in your web browser
  2. Accept or reject cookies in the Cookie Consent Banner (appears after 1.5 seconds)
  3. The splash screen appears showing a loading progress bar with personalized greeting
  4. First-time users: Enter the 8-character access code provided by the developer
  5. Complete the onboarding flow to set up your profile (optional but recommended)
  6. First-time setup: Install dependencies via Terminal (press Alt+T and run heyo install bugwizard dependencies)

Access Code Protection

BugWizard requires an 8-character access code to unlock the application. There are two types of codes:

License Code (permanent access)

  • Format: PLBW + 4 alphanumeric characters (e.g., PLBWA7K9)
  • Provides unlimited permanent access
  • No expiration or countdown
  • Enter in the main Access Code field

Trial Code (3-day access)

  • Format: TLBW + 4 alphanumeric characters (e.g., TLBWA7K9)
  • Provides full access for 3 days from activation
  • Trial codes are valid for 3 days from generation date
  • Access via the "🎟️ Try 3-Day Trial" button on the Access modal

To Unlock with License Code

  1. Enter the 8-character license code (starts with PLBW)
  2. Toggle password visibility using the 👁️ button
  3. Click "🚀 Unlock BugWizard"
  4. Once authenticated, access is remembered for future sessions

To Activate Trial

  1. Click "🎟️ Try 3-Day Trial" button on the Access modal
  2. Enter the 8-character trial code (starts with TLBW)
  3. Click "🚀 Activate Trial"
  4. Trial period begins immediately upon activation

Trial Mode Features

When in trial mode, a banner appears at the top showing:

  • Trial Mode indicator with 🎟️ icon
  • Days remaining (e.g., "3 days remaining")
  • Countdown timer in DD:HH:MM:SS format
  • "🚀 Upgrade" button to get full access

A floating 👑 Upgrade widget also appears for quick access to upgrade options.

Requesting Access Codes

Click the "📧 Contact Developer" link on the Access modal to send an email request via Gmail with a pre-filled template including your name, role, and company.

💡
Tip License codes provide permanent access, while trial codes are great for evaluation. Contact the developer to obtain either type.

⚙️ Initial Setup

Step 1: Install Dependencies (First-Time Only)

When you first launch BugWizard, a setup overlay will prompt you to install dependencies:

  1. Press Alt+T (or Cmd+T on Mac) to open the Terminal
  2. Run the command: heyo install bugwizard dependencies
  3. Wait for installation to complete
  4. This is a one-time setup process

Step 2: Configure API Token (Required)

Before using BugWizard's API features, you must configure your platform credentials:

For Azure DevOps:

  1. Click the ⚙️ Settings icon in the dock (bottom of screen)
  2. Select "🔑 API Authentication"
  3. Enter your Personal Access Token (PAT)
  4. Click "💾 Save Token"

For JIRA:

  1. Click the ⚙️ Settings icon in the dock
  2. Select "🔑 API Authentication" (will show JIRA fields if JIRA platform is selected)
  3. Enter your JIRA Domain (e.g., your-company.atlassian.net)
  4. Enter your Email address
  5. Enter your JIRA API Token
  6. Enter your Default Project Key (e.g., BUG, PROJ)
  7. Click "💾 Save JIRA Settings"
💡
Tip Generate a JIRA API token from Atlassian Account Settings.
⚠️
Important The PAT token is required for: Duplicate bug detection, Automatic retest comment submission, Bug status updates, Video/image attachment uploads, and QA Bug Dashboard.

Step 3: Set Up Your Profile (Recommended)

During the first-time onboarding, or via Settings:

  1. Enter your Full Name
  2. Enter your Email (choose from predefined domains or enter custom)
  3. Click "🚀 Get Started" or save from the profile editor

Step 4: Add Custom Apps (Optional)

  1. Click ⚙️ Settings → "📱 Custom Apps"
  2. Choose "➕ Add New App"
  3. Enter your application name → Click "➕ Next"
  4. Add markets (comma-separated) or click "Skip (No Markets)"
  5. Click "💾 Save App"
💡
Your custom apps will appear in the "Select App" dropdown with a 🔹 icon.

Step 5: Extend Default Apps (Optional)

Add additional markets to your default applications:

  1. Click ⚙️ Settings → "📱 Custom Apps"
  2. Choose "✨ Extend Default Apps"
  3. Select the default app from the dropdown
  4. Enter market name(s) (comma-separated for multiple)
  5. Click "➕ Add Market"

✅ Retester Mode

Developer's Chill Mode 🤩

Use this mode to submit retest reports as comments on existing bugs in Azure DevOps or JIRA.

Step-by-Step Process

1. Enter Bug ID

When in Retester Mode, a Bug ID input field appears inside the text area (top-left). Enter the work item ID (Azure DevOps) or issue key (JIRA, e.g., BUG-123) of the bug you're retesting.

2. Paste Retest Report

Paste your markdown-formatted retest report including Status (Passed/Failed), Description, Device Model, OS Version, App Version, Testing Environment, Retesting Date, Retested By, and Conclusion.

3. Parse and Submit

  1. Click "Parse Pasted Content" to extract retest details
  2. Review the parsed data
  3. Click "Add Retest Comment 📝" to submit
  4. Confirm the submission in the dialog

What Happens on Submission

The following occurs automatically via Azure DevOps REST API (or JIRA REST API if JIRA is selected):

Action Description
Comment Added A professionally formatted HTML comment is added to the bug's Discussion section (ADO) or Comments (JIRA)
Status Updated Passed → Bug state set to "Fixed" (ADO) or configured transition (JIRA)
Failed → Bug state set to "Failed" (ADO) or configured transition (JIRA)
Tags Updated Device-specific tags added (iPad/iPhone)
Work Item Opens Azure DevOps or JIRA opens in a new tab showing the updated bug
⚠️
Requirement PAT token must be configured with Write permissions for this feature to work.

🧠 Train AI for Your Project

BugWizard allows you to upload custom training data to make AI responses more relevant to your specific project.

What is AI Training Data?

Training data is project-specific context that gets injected into every AI conversation. This helps the AI understand:

  • Your project's terminology and naming conventions
  • Specific testing requirements and workflows
  • Known issues and edge cases
  • Team standards and best practices

How to Upload Training Data

Method 1: From Ozzy AI Sidebar

  1. Open Ozzy AI (🤖 icon in dock)
  2. Click "📄 Upload Training Data (.md / .json)" in the sidebar
  3. Select your training file
  4. The file will be loaded and shown in the sidebar with a 🧠 icon

Method 2: From Connect AI Provider Card

  1. Open Ozzy AI
  2. In the "Train AI for Your Project" card at the bottom
  3. Click "📄 Upload Training Data (.md / .json)"
  4. Select your training file

Supported File Formats

Format Extension Best For
Markdown.mdDocumentation, guides, structured text
JSON.jsonStructured data, configurations

Training Data Indicator

Once training data is loaded:

  • Sidebar: Shows 🧠 icon with filename (e.g., "project-guide.md")
  • Bottom Card: Shows ✓ with file size (e.g., "project-guide.md (15.2 KB loaded)")

Removing Training Data

  • Sidebar: Click the ✕ button next to the filename
  • Bottom Card: Click the ✕ button in the status area

How Training Data Works

When you send a message to the AI, BugWizard:

  1. Loads the base system prompt (Ozzy AI context)
  2. Appends your training data as "PROJECT-SPECIFIC TRAINING DATA"
  3. Sends the combined context with your message

This ensures every response is informed by your project-specific information.

💡
Best Practices for Training Data
  • Keep it concise – Include only relevant information
  • Use clear headings – Help AI understand document structure
  • Include examples – Show expected formats and outcomes
  • Update regularly – Keep training data current with project changes
  • Use markdown – Well-formatted .md files work best

Storage

  • Training data is stored in browser localStorage
  • Key: aiTrainingData (content)
  • Key: aiTrainingFileName (display name)
  • Data persists across browser sessions

🔧 Troubleshooting

Common Issues

"Access Code Invalid" Error

❌
Cause: Incorrect access code format or invalid code entered.
Solution: Verify the access code is exactly 8 characters. Trial codes start with TLBW (e.g., TLBWA7K9). License codes start with PLBW (e.g., PLBWA7K9). Contact the developer for a valid trial or license code.

"Invalid Code or Expired" Error

⏰
Cause: Trial code has expired (trial period is 3 days).
Solution: Request a new trial code from the developer, or purchase a license code for permanent access.

"PAT Token not configured" Error

🔑
Cause: PAT token is not saved in settings.
Solution: Go to Settings → API Authentication, enter your PAT token, click "💾 Save Token".

Parse button is disabled

⚠️
Cause: Required fields not selected.
Solution: Ensure you've selected Issue Type, Application, and Market (if required).

Duplicate detection returns no results

Possible causes:

  • Keywords from title are too common
  • No matching bugs in your configured project
  • PAT token lacks read permissions
  • Network connectivity issues

Solution: Check browser console (F12) for API errors. Verify PAT has Work Items Read scope.

Retest comment submission fails

Solution: Verify PAT permissions include "Work Items - Read & Write". Confirm the Bug ID exists and you have access.

Video compression fails

Solution: Use a supported format (MP4, WebM, MOV, AVI). Try a smaller video file. Use Chrome or Edge for best compatibility.

Ozzy AI says "API key required"

Solution: Select an AI provider, enter your API key, click "🚀 Connect".

AI Training Data not loading

📄
Possible causes: File format not supported (must be .md or .json), file is empty or corrupted, browser localStorage is full.
Solution: Verify file has .md or .json extension. Check file contains valid content. Try clearing browser cache if localStorage is full.

AI responses don't reflect training data

🧠
Possible causes: Training data was not uploaded correctly, training data was cleared, API key expired or invalid.
Solution: Check sidebar for 🧠 icon with filename. Re-upload training data if not showing. Verify API key is still connected.
